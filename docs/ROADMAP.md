# 모이다 (Moida) 개발 로드맵

> 한 번의 모임을 위한, 가장 가벼운 운영 도구.

---

## 개요

**30명 이내 소규모 모임 주최자**를 위한 단일 이벤트 페이지 기반 통합 운영 도구입니다.
카카오톡·구글폼·더치트 등 흩어진 도구를 하나로 묶어 도구 전환 비용을 제거합니다.

| 핵심 기능          | 한 줄 설명                                             |
| ------------------ | ------------------------------------------------------ |
| 공지사항           | 단톡방에 묻히지 않는 모임 전용 공지 + 이메일 자동 발송 |
| 참여자 관리 (RSVP) | 참석 / 불참 / 미정 3종 상태 + 자동 멤버 등록           |
| 정산               | 총액 N등분 자동 계산 + 스냅샷 고정 방식                |
| 카풀               | 차량 등록 + 드롭다운 기반 탑승자 수동 배정             |

---

## 트랙 구조 (병렬 개발 전략)

Phase 1 완료 시점부터 두 개의 트랙을 **동시에 진행**합니다.
공통 기반 트랙(DB 스키마)은 두 트랙 모두에 영향을 미치므로 가장 먼저 착수합니다.

```
                         ┌─ [트랙 A] 프론트엔드 ─────────────────┐
Phase 0 ──> Phase 1 ──┤                                          ├──> Phase 6 (폴리싱)
                      │  └─ [트랙 B] 어드민 ──────────────────┤
                      │
                      └─ [공통] Phase 3 (DB 스키마) — 두 트랙 모두에 선행/병렬 가능
```

| 트랙        | 담당 영역                                     | 해당 Phase         |
| ----------- | --------------------------------------------- | ------------------ |
| 공통 기반   | DB 스키마, Supabase 초기 설정, 타입 자동 생성 | Phase 3            |
| 트랙 A      | 프론트엔드 사용자 화면 및 핵심 비즈니스 기능  | Phase 2A → Phase 4 |
| 트랙 B      | 어드민 읽기 전용 운영 도구                    | Phase 2B → Phase 5 |
| 통합 폴리싱 | 두 트랙 완료 후 최종 출시 품질 마감           | Phase 6            |

> Phase 1~6 완료 후 아래 시나리오가 매끄럽게 흐르면 출시 가능 상태로 판단합니다.

|  #  |  사용자  | 행동                                        | 검증 포인트                             |
| :-: | :------: | ------------------------------------------- | --------------------------------------- |
|  1  | 비로그인 | 메인 진입 → 이벤트 목록·상세 조회           | 기본 정보 열람 가능                     |
|  2  |  계정 A  | 회원가입·로그인 → 이벤트 생성               | Owner 자동 지정                         |
|  3  |  계정 B  | 공개 목록에서 발견 → "참여하기"             | 자동 멤버 등록 + RSVP "참석"            |
|  4  |    A     | B를 Co-host 지명                            | B에게 공지 작성 권한 부여               |
|  5  |    B     | 공지 작성                                   | A·B 이메일 수신 확인                    |
|  6  |    B     | 카풀 차량 추가 (좌석 2) → A 배정            | 드롭다운 배정 정상 동작                 |
|  7  |    A     | 정산 생성 (30,000원, 참석 2명)              | 1인당 15,000원 표시                     |
|  8  |    A     | 이벤트 삭제                                 | 종속 데이터 개수 다이얼로그 → 삭제 완료 |
|  9  | 어드민 C | `/admin/users`에서 A·B 목록 확인 + "A" 검색 | 검색 결과 정확히 1건                    |
| 10  | 어드민 C | `/admin/events/[id]` 진입 (비멤버)          | 4섹션 열람 가능, 작성/수정 버튼 미노출  |

> 전체 시나리오를 Playwright MCP로 자동화하여 회귀 검증에 활용합니다.
