# 모임 이벤트 관리 웹 MVP PRD

> 한 번의 모임을 위한, 가장 가벼운 운영 도구.

---

## 1. 제품 개요

### 1.1 한 줄 정의

**한 번의 모임을 위한, 가장 가벼운 운영 도구.**

### 1.2 해결하려는 문제

수영·헬스·독서·친구 모임 등 **30명 이내 소규모 모임 주최자**는 한 번의 모임을 운영하기 위해 보통 4개의 도구를 동시에 사용합니다.

| 사용 도구     | 용도           | 불편함                               |
| ------------- | -------------- | ------------------------------------ |
| 카카오톡      | 공지 전달      | 채팅 위로 떠밀려 못 보는 사람이 생김 |
| 구글폼        | 참석 명단 수집 | 명단·이벤트 정보가 따로 흩어짐       |
| 더치트 / 토스 | 정산           | 누가 왔는지 다시 세야 함             |
| 카톡 협의     | 카풀 조율      | 좌석 충돌, 누락 빈번                 |

→ **단일 이벤트 페이지 한 곳**에 공지·참여자·정산·카풀 4기능을 통합하여 도구 전환 비용을 제거합니다.

### 1.3 타겟 사용자 (페르소나)

#### 페르소나 ① 동호회 회장 박상철 (40대)

- 자전거 동호회 25명, 월 1~2회 라이딩
- 핵심 고통: "지난주 누가 왔는지 기억이 안 난다"
- 니즈: 참석 기록 + 회비 정산을 한 번에

#### 페르소나 ② 친구 모임 총무 김지은 (20대 후반)

- 대학 동기 8명, 분기 1회 모임
- 핵심 고통: "정산이 매번 내 몫이라 부담"
- 니즈: 1/N 자동 계산 + 입금 확인 흔적

#### 페르소나 ③ 일반 참가자 이민재 (30대)

- 여러 모임에 속한 참여자
- 핵심 고통: "공지가 단톡방 위로 올라가서 못 본다"
- 니즈: 모임별로 깔끔하게 정리된 공지/일정

### 1.4 성공의 정의 (3개월 후)

- 모임 주최자가 카카오톡 + 구글폼 + 더치트 조합을 **이 서비스 하나로** 대체했다고 말한다
- "한 모임을 만들고 → 사람을 모으고 → 공지 보내고 → 정산하고 → 카풀 맞추는" 시나리오가 단일 페이지에서 매끄럽게 흐른다

## 2. 사용자 (역할) 모델

### 2.1 핵심 원칙

> **"주최자"와 "참가자"는 사람의 속성이 아니라, 이벤트별 역할이다.**

- 누구나 자유롭게 회원 가입 (이메일 또는 Google)
- **회원이면 누구나 이벤트를 생성**할 수 있고, 생성하는 순간 그 이벤트의 **주최자(Owner)** 가 됨
- 같은 사람이 다른 이벤트에서는 **참가자(Participant)** 가 될 수도 있음

### 2.2 이벤트 내 역할 3종

| 역할                      | 부여 방식                                 | 핵심 권한                                                |
| ------------------------- | ----------------------------------------- | -------------------------------------------------------- |
| **Owner (원본 주최자)**   | 이벤트 생성자에게 자동 부여, 이벤트당 1명 | 이벤트 수정/삭제, Co-host 지명·해임, 공지·정산·카풀 관리 |
| **Co-host (공동 주최자)** | Owner가 멤버 중 지명                      | 공지·정산·카풀 관리, 멤버 강퇴 (이벤트 삭제 불가)        |
| **Participant (참가자)**  | 멤버 등록 시 기본 부여                    | 본인 RSVP 변경, 멤버 명단/공지/정산/카풀 조회            |

### 2.3 공개·접근 정책

| 콘텐츠                                                | 비로그인         | 회원 (비멤버) | 멤버                  |
| ----------------------------------------------------- | ---------------- | ------------- | --------------------- |
| 이벤트 목록                                           | 조회 가능        | 조회 가능     | 조회 가능             |
| 이벤트 기본 정보 (제목/일시/장소/설명/정원/참석자 수) | 조회 가능        | 조회 가능     | 조회 가능             |
| 이벤트 생성                                           | 불가 (가입 유도) | 가능          | 가능                  |
| 참여하기 (RSVP)                                       | 불가 (가입 유도) | 가능          | 해당 없음 (이미 멤버) |
| 멤버 명단                                             | 불가             | 불가          | 조회 가능             |
| 공지사항                                              | 불가             | 불가          | 조회 가능             |
| 정산 상세                                             | 불가             | 불가          | 조회 가능             |
| 카풀 상세                                             | 불가             | 불가          | 조회 가능             |

### 2.4 이벤트 참여 진입점 (2가지)

1. **공개 발견 경로**: 이벤트 목록에서 발견 → "참여하기" 클릭 → (비로그인이면 가입 유도) → 자동 멤버 등록
2. **초대 링크 경로**: 주최자가 카톡으로 공유한 링크 클릭 → (비회원이면 가입 유도) → 자동 멤버 등록

## 3. 사용자 여정

```
[비로그인 방문]
    ↓ 이벤트 목록 / 상세 (기본 정보) 열람 가능
    ↓
[이벤트 발견] ───────────────┐
    ↓ "참여하기" 클릭          │
    ↓ (회원 가입 또는 로그인)  │     [초대 링크 클릭]
    ↓                          │           ↓
    └──────────┬───────────────┘     (회원 가입 또는 로그인)
               ↓                            ↓
        [자동 멤버 등록 + RSVP "미정"] ←────┘
               ↓
        [이벤트 상세 페이지 진입]
               ↓
   ┌───────────┼────────────┬─────────────┐
   ↓           ↓            ↓             ↓
 [공지 읽기] [RSVP 변경]  [정산 확인]   [카풀 확인]
                ↓
         (이메일 알림 수신: 새 공지 등록 시)
                ↓
         [모임 종료 후 일시 24시간 경과]
                ↓
         RSVP 잠금 / "지난 이벤트" 표시
```

### 주최자 측 여정

```
[로그인]
    ↓ "이벤트 만들기"
    ↓ 제목/일시/장소/설명/정원 입력
    ↓
[이벤트 생성 완료 + 자동 Owner 지정]
    ↓
[초대 링크 발급] → 카톡 등으로 공유
    ↓
[멤버 모이는 중] ───────────────────────┐
    ↓                                    │
[공지 작성] → 멤버 전원 이메일 자동 발송 │
    ↓                                    │
[Co-host 지명] (선택)                    │
    ↓                                    │
[정산 생성] (총액 입력 → 1/N 자동 계산)  │
    ↓                                    │
[카풀 추가] (차량 + 운전자 + 좌석 수)    │
    ↓                                    │
[탑승자 배정] (드롭다운 선택) ───────────┘
    ↓
[모임 종료] → (선택) 이벤트 삭제 (즉시 삭제 + 종속 데이터 개수 확인)
```

## 4. 핵심 기능

### 4.1 이벤트 (공통 기반)

| 항목                | 내용                                                                       |
| ------------------- | -------------------------------------------------------------------------- |
| 생성 권한           | 모든 회원                                                                  |
| 자동 역할           | 생성자 = Owner                                                             |
| 필수 필드           | 제목(≤80자), 일시(시작 시각), 장소(≤200자), 설명(≤2000자), 최대인원(1~200) |
| 과거 일시           | 등록 허용 (백필 시나리오 지원)                                             |
| 일시 24시간 경과 후 | "지난 이벤트" 표시 + RSVP 잠금                                             |
| 정원 초과 시        | 신규 "참석" 차단 (불참/미정은 허용)                                        |
| 삭제                | 즉시 삭제(Hard delete), 다이얼로그에 종속 데이터 개수 표시                 |

### 4.2 공지사항

| 항목           | 내용                                         |
| -------------- | -------------------------------------------- |
| 작성 권한      | Owner, Co-host                               |
| 열람 권한      | 모든 멤버                                    |
| 수정/삭제      | 작성자 본인 + Owner                          |
| 이메일 발송    | 작성 시 자동, 멤버 전원에게 (RSVP 상태 무관) |
| 수정 시 재발송 | 없음                                         |
| **MVP 제외**   | 핀 고정, 중요 표시, 댓글, 반응(좋아요 등)    |

### 4.3 참여자 관리 (RSVP)

| 항목                | 내용                                                        |
| ------------------- | ----------------------------------------------------------- |
| 승인 방식           | **자동 승인** (별도 승인 단계 없음)                         |
| RSVP 옵션           | 참석 / 불참 / 미정 (기본값: 미정)                           |
| RSVP 잠금           | 이벤트 일시 경과 후 변경 불가                               |
| 본인 직접 참여 취소 | Participant·Co-host 가능 / Owner 불가                       |
| 멤버 강퇴           | Owner·Co-host 가능 (단, 자기 자신 강퇴 불가)                |
| 초대 링크           | 이벤트당 1개 발급, 만료 없음, **비활성화/재발급 토글 가능** |

### 4.4 정산

| 항목          | 내용                                                           |
| ------------- | -------------------------------------------------------------- |
| 정산 모델     | **총액 N등분 (단일 금액)**                                     |
| N (정산 대상) | RSVP가 "참석"인 멤버 전원 (역할 무관)                          |
| N 산정 시점   | **정산 생성 시점에 스냅샷 고정** (이후 RSVP 변경은 반영 안 됨) |
| 1인당 계산    | `Math.ceil(총액 / N)` — **올림 처리**, 잔돈은 주최자 부담      |
| 총액 범위     | 1원 이상 ~ 1억 이하                                            |
| 수정/삭제     | Owner, Co-host                                                 |
| 이메일 발송   | 없음 (필요 시 공지로 별도 안내)                                |
| 다중 정산     | 한 이벤트에 여러 개 가능 (예: 점심값 / 회비 분리)              |
| 실제 송금     | **영구 제외** (계산기 역할만)                                  |
| **MVP 제외**  | 항목별 정산 (v1.1)                                             |

### 4.5 카풀 매칭

| 항목                | 내용                                                                            |
| ------------------- | ------------------------------------------------------------------------------- |
| 매칭 방식           | **주최자 수동 배정**                                                            |
| 차량 입력 정보      | 운전자(멤버 중 선택), 좌석 수(1~10, 운전자 미포함 동승 인원), 출발지, 출발 시각 |
| 탑승자 배정 UI      | **드롭다운 선택** (드래그앤드롭 없음, 모바일/데스크톱 동일)                     |
| 중복 탑승           | 불가 (한 사람은 한 차량만, 다른 차 배정 시 이전 자동 해제)                      |
| 좌석 초과           | 추가 배정 차단                                                                  |
| 운전자 본인 탑승    | 차단 (이미 운전 중)                                                             |
| RSVP "불참" 전환 시 | 카풀 배정 자동 해제                                                             |
| 운전자 강퇴 시      | 해당 차량 자동 삭제                                                             |
| **MVP 제외**        | 멤버 셀프 탑승 신청, 자동 매칭                                                  |

## 5. 화면 구조 (페이지 라우트)

| 경로                | 페이지                           | 비로그인 접근 | 비고                                      |
| ------------------- | -------------------------------- | :-----------: | ----------------------------------------- |
| `/`                 | 홈 (공개 이벤트 목록 포함)       |       ✓       |                                           |
| `/events`           | 이벤트 목록                      |       ✓       | 공개 이벤트 전체                          |
| `/events/[id]`      | 이벤트 상세 (단일 페이지)        |       △       | 비로그인은 기본 정보만, 멤버는 4기능 전체 |
| `/events/new`       | 이벤트 생성                      |       ×       | 가입 유도                                 |
| `/events/[id]/edit` | 이벤트 수정                      |       ×       | Owner 전용                                |
| `/invite/[token]`   | 초대 링크 진입                   |       ✓       | 비회원은 가입 유도                        |
| `/auth/*`           | 로그인·회원가입·비밀번호 찾기 등 |       ✓       | 기존 구현 활용                            |
| `/profile`          | 내 프로필                        |       ×       |                                           |

### 이벤트 상세 페이지 (`/events/[id]`) 구성

> **단일 페이지 안에서 4기능이 섹션으로 모두 보임.**

```
┌─────────────────────────────────────────┐
│ [이벤트 기본 정보]                       │ ← 모두 접근 가능
│ 제목 / 일시 / 장소 / 설명 / 정원 / 참석수│
├─────────────────────────────────────────┤
│ [멤버 명단 섹션]                         │ ← 멤버만
│ - RSVP 상태별 그룹화                     │
│ - 본인 RSVP 변경 / 강퇴 / 본인 취소     │
├─────────────────────────────────────────┤
│ [공지사항 섹션]                          │ ← 멤버만
│ - 최신순 리스트                          │
│ - 작성 버튼 (Owner·Co-host)              │
├─────────────────────────────────────────┤
│ [정산 섹션]                              │ ← 멤버만
│ - 총액 / 인원 / 1인당 표시               │
│ - 생성·수정·삭제 (Owner·Co-host)         │
├─────────────────────────────────────────┤
│ [카풀 섹션]                              │ ← 멤버만
│ - 차량 카드 + 탑승자 드롭다운            │
│ - 차량 추가 (Owner·Co-host)              │
└─────────────────────────────────────────┘
```

## 9. 확정된 결정사항 (검토 라운드 결과)

## 12. 출시 검증 시나리오 (E2E)

> 아래 시나리오 전체가 매끄럽게 흐르면 출시 가능 상태로 판단합니다.

1. **비로그인** 상태로 메인 페이지 진입 → 이벤트 목록·상세 기본 정보까지 열람 가능 확인
2. **계정 A** 회원가입 후 로그인 → 이벤트 생성 (제목·일시·장소·설명·정원 입력) → Owner 자동 지정 확인
3. **계정 B** 회원가입 → 공개 목록에서 A의 이벤트 발견 → "참여하기" → 자동 멤버 등록 + RSVP "참석"으로 변경
4. **A**가 B를 **Co-host로 지명** → B에게 공지 작성 권한 부여 확인
5. **B**가 공지 작성 → A와 B 모두에게 **이메일 수신 확인**
6. **B**가 카풀 차량 추가 (운전자 = B, 좌석 2) → A를 탑승자로 드롭다운 배정
7. **A**가 정산 생성 (총액 30,000원) → 참석 멤버 2명 기준 1인당 15,000원 표시 확인
8. **A**가 이벤트 삭제 → 다이얼로그에 종속 데이터 개수 (공지 1, 정산 1, 카풀 1) 표시 → 확인 후 삭제 완료

## 부록 A. 데이터 모델 (개념 수준)

> 실제 컬럼 정의·SQL은 Phase 1 구현 단계에서 마이그레이션 파일로 확정합니다. 본 문서에서는 개념 수준의 테이블과 관계만 정의합니다.

| 테이블                        | 핵심 필드                                                                           | 관계                                              |
| ----------------------------- | ----------------------------------------------------------------------------------- | ------------------------------------------------- |
| `profiles`                    | id, email, name                                                                     | Supabase Auth와 1:1                               |
| `events`                      | id, title, starts_at, location, description, capacity, owner_id                     | → profiles (owner)                                |
| `event_members`               | id, event_id, user_id, role(owner/co-host/participant), rsvp_status(참석/불참/미정) | → events, profiles                                |
| `invite_tokens`               | id, event_id, token, is_active                                                      | → events                                          |
| `notices`                     | id, event_id, author_id, title, body, created_at                                    | → events, profiles                                |
| `notice_email_logs`           | id, notice_id, recipient_user_id, status, error_message                             | → notices, profiles                               |
| `settlements`                 | id, event_id, title, total_amount, snapshot_member_count, per_person_amount         | → events                                          |
| `settlement_snapshot_members` | id, settlement_id, user_id                                                          | → settlements, profiles (정산 시점 참석자 스냅샷) |
| `carpools`                    | id, event_id, driver_id, seat_count, origin, depart_at                              | → events, profiles                                |
| `carpool_passengers`          | id, carpool_id, user_id                                                             | → carpools, profiles                              |
